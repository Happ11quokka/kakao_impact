# P — Plan: 감정분석 Recap UX 개편

## 목표

감정분석 화면을 “숫자/막대 분석” 중심에서 “기간별 감정 회고를 고르고 기록을 다시 보는” 흐름으로 바꾼다.

## 사용자 요구

1. 감정분석 내비게이션의 감정요약 카드에서 원석이 함/보관함 안에 들어있는 느낌이 없어 직관성이 낮다.
2. 감정패턴 시각화는 아코디언 형태 UI로 접고 펼칠 수 있어야 한다.
3. 기존 “계열별 세부 감정 분석” 대신 주간/월간 감정 recap 같은 여러 테마 슬라이드를 가로로 넘기며 볼 수 있어야 한다.
4. 리캡 테마를 선택하면 관련 기록이 텍스트와 사진을 포함해 보여야 한다.
5. 기간 선택은 주간/월간/직접이며, 직접은 사용자가 시작일/종료일을 직접 선택해야 한다.
6. 개발은 PABCD 하네스 기준으로 기록한다.

## 구현 전략

- `Analysis.tsx` 안에서 기간 필터를 `weekly | monthly | custom` + `customStart/customEnd`로 확장한다.
- `AnalysisItem`을 사진 URL까지 담도록 확장하고, `ChatbotRecordDto`와 같은 날짜 기반으로 기록 텍스트/사진을 연결한다.
- 감정요약 카드는 작은 원석 보관함/트레이 비주얼을 추가해 “요약된 감정 원석들이 한 곳에 담긴다”는 은유를 강화한다.
- 감정패턴 시각화는 기본 펼침 상태의 아코디언으로 만든다.
- 세부 감정 분석 섹션은 제거하고 `recapThemes` 배열 기반의 가로 스냅 슬라이더 + 선택된 테마의 기록 리스트로 대체한다.
- 기존 시간대별 분포/질문/리포트/케어 카드는 최대한 유지하되 리캡 UX와 중복되는 제목은 정리한다.

## 테스트 계획

- `Analysis.test.ts`에 순수 헬퍼 테스트를 먼저 추가한다.
- 테스트 대상:
  - 직접 기간은 start/end 사이 날짜만 포함한다.
  - 같은 날짜의 chatbot record가 item에 텍스트와 사진 URL로 연결된다.
  - 리캡 테마는 데이터가 있을 때 “가장 또렷한 감정”, “사진이 있는 순간”, “돌아볼 기록” 같은 카드와 관련 기록을 만든다.
- RED 확인 후 구현한다.
- `npm test -- Analysis.test.ts`, `npm run build`로 검증한다.
- 브라우저에서 `/analysis` 진입, 개발용 로그인 후 UI/콘솔을 확인한다.

## 범위 밖

- 백엔드 API 변경 없음.
- 실제 월간 PDF/공유 이미지 생성 없음.
- 장기 통계/전월 대비 계산은 이번 범위에서 카드 문구 수준으로만 처리.
