# A — Audit: 감정분석 Recap UX 개편

## 현재 구조 확인

- 대상 파일: `2_Ulog/frontend/src/routes/Analysis.tsx`
- 현재 화면 구조:
  1. 헤더 + 기간 탭
  2. 감정 요약 카드
  3. 감정 패턴 시각화 막대
  4. 계열별 세부 감정 분석
  5. 시간대별 분포
  6. 자기인지 질문 recap
  7. 주간/월간 리포트
  8. 케어/Recap 카드

## 발견한 구현 자산

- 이미 `Period = 'weekly' | 'monthly' | 'custom'` 타입이 있으나 custom은 “최근 2주”로 하드코딩되어 있음.
- `api.ts` mock API가 있어 로컬 개발에서 기록/원석/필드 데이터를 볼 수 있음.
- `GemStone` 컴포넌트가 있어 요약 카드에 원석 비주얼을 재사용 가능.
- 기존 테스트는 `Calendar.test.ts` 하나뿐이므로 `Analysis.test.ts`를 새로 추가하는 방식이 가장 작다.

## 리스크

- `Analysis.tsx`가 700줄 이상이라 무리한 분리는 충돌 위험이 있다. 이번에는 순수 헬퍼만 export하고 UI는 한 파일 안에서 최소 변경한다.
- `gems`와 `chatbotRecords` 연결 키가 명확한 ID가 아니라 날짜 기반 보강이므로 같은 날짜 기록이 여러 개일 때 완벽한 매칭은 아니다. 기존 코드도 날짜 기반 텍스트 연결을 사용하므로 이번 범위에서는 동일 패턴을 유지하되 사진 URL도 함께 가져온다.
- `custom` 기간 input은 모바일 브라우저별 날짜 선택 UI가 다르다. 최소 접근성으로 label/aria를 둔다.
- 이전 턴에서 복구한 로컬 개발 로그인 변경이 아직 커밋되지 않은 상태라 이번 diff에 같이 보인다. Done에 별도 항목으로 명시한다.

## 결정

- Build 진행 가능.
- TDD 범위는 “기간 필터/기록 연결/리캡 테마 생성” 순수 로직으로 잡는다.
- 스타일 회귀는 브라우저 수동 확인으로 보완한다.
