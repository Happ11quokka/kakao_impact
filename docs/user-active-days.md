# 사용자 활동 일수 분포 (첫 접속 후 며칠 사용했는가)

> 집계 시각: 2026-05-28 · 데이터 소스: Railway `intelligent-wholeness` / Postgres (production)
> 분석 기간: 2026-04-23 ~ 2026-05-28 (KST, 첫 이벤트 ~ 마지막 이벤트)

## 요약

- **등록 사용자 42명** 중 **13명(31.0%)** 은 추적 가능한 활동이 0일 — 가입했지만 실제 사용 흔적 없음.
- 한 번이라도 사용한 **활동 사용자 29명** 중 **12명(41.4%)** 은 단 하루만 사용하고 다시 오지 않음.
- 5·6·7·8일 활동 사용자는 **0명** — 4일 이후 뚝 끊겼다가, 충성 사용자 5명만 9일 이상 활동.

## 집계 방법

- **활동 일수** = 해당 사용자가 이벤트(`events`)를 1건 이상 발생시킨 **서로 다른 KST 날짜(달력일)의 수**. "첫 접속 후 N일 사용" = 활동한 날이 N일.
- **사용자 식별**: `events.user_id` 우선, 없으면 `anon_user_links` 로 익명 `anonId` → 사용자 매핑(익명 stitching). 둘 다 없으면 비로그인 익명 방문자로 분류.
- **제외**: 운영자 대시보드 경로(`/ops/%`) 트래픽 제외 (ops 대시보드 분석 쿼리와 동일 규칙). `OPS_ALLOWED_KAKAO_IDS` 환경변수는 미설정이라 특정 운영자 user 제외는 적용 안 됨.
- **날짜 경계**: `occurred_at AT TIME ZONE 'Asia/Seoul'` 기준 KST 달력일.
- 데이터 규모: 등록 사용자 42명, 이벤트 3,053건, 익명 링크 31건.

### 재현 쿼리

```sql
WITH owner AS (
  SELECT
    COALESCE(e.user_id, l.user_id) AS uid,
    (e.occurred_at AT TIME ZONE 'Asia/Seoul')::date AS kst_day
  FROM events e
  LEFT JOIN anon_user_links l
    ON l.anon_id = (e.props->>'anonId') AND e.user_id IS NULL
  WHERE (e.props->>'path' IS NULL OR e.props->>'path' NOT LIKE '/ops/%')
),
per_user AS (
  SELECT u.id, COUNT(DISTINCT o.kst_day) AS active_days
  FROM users u
  LEFT JOIN owner o ON o.uid = u.id
  WHERE u.deleted_at IS NULL
  GROUP BY u.id
)
SELECT active_days, COUNT(*) AS users
FROM per_user GROUP BY active_days ORDER BY active_days;
```
