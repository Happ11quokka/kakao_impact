## 📐 중앙값(median) 체류 계산법 — 공식 정리

대시보드가 쓰는 "평균"과, 이 리포트에 추가한 "중앙값"은 **같은 원천 데이터**(`page.dwell` 이벤트의 `durationMs`)에서 나온다.

**1) 원천 데이터**
`events` 테이블에서 `event_type = 'page.dwell'` 행. 각 행의 `props->>'durationMs'`(밀리초)가 한 번 페이지에 머문 시간이다. (`page.dwell`은 사용자가 페이지를 떠날 때 프론트가 쏘는 이벤트.)

**2) 전처리 (대시보드와 동일)**
```sql
초 = NULLIF((props->>'durationMs')::numeric, 0) / 1000
```
- `NULLIF(.., 0)` → `durationMs`가 정확히 0인 표본은 **NULL로 빼버림**(평균·중앙값 계산에서 제외). 그래서 "체류 0초 비율"이 모두 0%로 나온 것.
- `/ 1000` → 밀리초를 초로 변환.

**3) 평균 (대시보드 표의 "체류")**
```sql
AVG(초)
```
산술평균이라 **탭 방치(최대 51~85시간) 한두 건**이 통째로 끌어올려 홈 8648초 같은 비현실적 값이 나온다.

**4) 중앙값 (이 리포트의 "중앙값체류")**
```sql
PERCENTILE_CONT(0.5) WITHIN GROUP (ORDER BY 초)
```
- 경로별로 표본을 **오름차순 정렬한 뒤 정확히 가운데 값**을 취함.
- 표본 수가 짝수면 가운데 두 값을 **선형 보간**(continuous)해서 평균.
  - `PERCENTILE_CONT` = 보간(연속), `PERCENTILE_DISC` = 실제 존재하는 값 그대로 → 여기서는 보간 버전 사용.
- 이상치에 흔들리지 않음. 예: 홈 370개 표본 중 **절반이 28.8초 이하**라는 뜻 → "보통 사용자의 실제 체류"는 이 값이 정답.
- p75 / p90도 같은 함수에 `0.75` / `0.90`을 넣은 것.

> **요약**: 평균은 이상치에 휘둘리고, 중앙값은 "전형적인 사용자"를 보여준다. 체류처럼 일부 값이 극단적으로 큰 지표는 **중앙값**으로 봐야 한다.

## 🧭 종합 인사이트

1. **리텐션은 의외로 견고** — 로그인 사용자 57%가 2일 이상, 21%가 4일 이상 재방문. 소규모지만 충성 코어 존재 (이인명·박수진·임동현이 5일 풀 활동).
2. **감정확정 vs 회고는 사용자별로 갈림** — 황수지(감정확정 10/회고 0), 류혜경·박수진(회고 4)처럼 두 기능 사용자가 분리됨. 회고는 전체 11건으로 매우 소수만 사용.

4. **계측 갭** — `data-track`이 5개 요소에만 있어 "감정 기록하기" 같은 핵심 전환 버튼 클릭이 안 잡힘. 추적 태그 확대 권장.
