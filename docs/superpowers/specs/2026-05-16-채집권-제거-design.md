# 채집권 시스템 제거 → 오늘 기록 카운터 전환

**날짜**: 2026-05-16  
**범위**: 챗봇(`2_avoha/ai/chatbot/main.py`), 백엔드(`2_avoha/backend/app/routes/ops.py`)  
**제외 범위**: 프론트엔드, DB 테이블(유지)

---

## 배경 및 목적

아보하 챗봇은 사용자가 하루 5개까지만 감정 원석을 채집할 수 있도록 `collection_tickets` 테이블과 인메모리 `user_count` dict를 이용한 채집권 시스템을 운영해왔다. 이 시스템을 **완전히 제거**하고, 대신 "오늘 몇 번 기록했는지"를 카운트하는 방식으로 전환한다.

**제거 이유**: 채집권 한도가 사용자의 감정 기록을 막는 진입 장벽이 됨.  
**새 방향**: 무제한 기록 허용, 완료 메시지에서 "오늘 N번째 원석이에요!" 형태로 누적 카운트 표시.


### 3. 추가할 함수

```python
def _db_get_today_count(user_id: str) -> int:
    """오늘(KST) 채집한 원석 수 (일상기록 제외). 에러 시 0 반환."""
    if not RAILWAY_DATABASE_URL:
        return 0
    today = _today_kst()
    try:
        conn = psycopg2.connect(RAILWAY_DATABASE_URL)
        cur = conn.cursor()
        cur.execute(
            """
            SELECT COUNT(*) FROM chatbot
            WHERE user_id = %s
              AND gem != '일상기록'
              AND (created_at AT TIME ZONE 'Asia/Seoul')::date = %s
            """,
            (user_id, today),
        )
        count = cur.fetchone()[0]
        cur.close()
        conn.close()
        return int(count)
    except Exception as e:
        print(f"[_db_get_today_count error] {e}")
        return 0
```

- `save_gem`은 background_tasks로 실행되므로 `_db_get_today_count`는 저장 직전에 호출해 "이 저장 후 N번째"가 되도록 count + 1로 표시.
- 에러 시 0 반환 → "오늘 첫 번째 원석이에요!"처럼 fallback.
